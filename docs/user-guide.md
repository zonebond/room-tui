## 前言

### 这份手册是什么

Room 装在您的电脑上，围绕**当前打开的项目目录**工作。配置好模型后，您可以：

- 用自然语言完成开发相关问答与辅助操作  
- 按样例模板派生结构化技术文档  
- 在界面中查看任务进度与文档大纲  

密钥与连接配置保存在本机用户目录，**不会**写入项目源码。

## 产品特色一览

Room 不是「网页聊天框搬进终端」，而是**嵌在项目目录里的工程助手**。下面几项是日常最能感知的差异：

HTML 交付页为 **6 块 · 双列**：上四行为体验，**末行一整行**为套件（paper-derived · oob-divzero，带轻量「套件」区分样式）。

| 特色 | 您会感受到什么 |
|------|----------------|
| **本地 · 以项目为中心** | 在哪个文件夹启动，就围绕哪个项目；密钥在本机用户目录 |
| **发送即开新一页** | 问题顶到对话区上方，回复从下面展开 |
| **长回复自动跟读** | 跟着最新输出走；上翻历史不会被强拽 |
| **忙时消息排队** | 上一轮未结束再发送也不会丢 |
| **paper-derived（套件）** | 样例 → `/new`；中断后 `/continue`（详见 [4.1.2](#412-随时中断随时-continue-恢复)） |
| **oob-divzero（套件）** | C 越界 / 除零：召回 → AI 判定 → 可选验证（详见 [4.2](#42-oob-divzero越界与除零扫描)） |

## 示例集 · 可以直接照着试

> 下列句子可直接粘贴到输入框（请先 `cd` 到真实项目再 `room`）。效果随模型与项目内容而异，用来熟悉交互即可。  
> 文档生成与 C 扫描的完整流程见 [第 4 章](#4-room-套件)。

### 对话示例

**摸清项目**

```text
用三五句话介绍当前项目：它是做什么的、主要目录有哪些、我该先从哪看起？
```

**对着报错问**

```text
我刚遇到这段报错，请用通俗话解释原因，并给出 2～3 个最可能的排查步骤：

（此处粘贴报错全文）
```

**对着具体文件问**（把路径换成您的）

```text
请阅读 src/ 下与启动入口相关的文件，说明程序是怎么启动的，用条目列出关键函数。
```

**改之前先问清楚**

```text
如果要给 README 增加「快速开始」一节，你会改哪些文件？先列计划，不要直接改文件。
```

### 五分钟体验路径

| 步骤 | 做什么 |
|------|--------|
| 1 | 终端：`cd 项目路径` → `room` |
| 2 | 若尚未配模型：`room setup` 或 **Ctrl+M** |
| 3 | 发送上面「摸清项目」那一句，观察**问题顶上、回复展开** |
| 4 | 回复较长时，试着**向上滚**再滚回来——应不会乱跳 |
| 5 | 有长折叠块时 **双击** 展开 |
| 6 | **Ctrl+B** 收起 / 展开右侧栏 |
| 7 | （可选）按 [4.1](#41-paper-derived模板派生文档) 跑一轮文档生成 |
| 8 | （可选，有 C 代码时）按 [4.2](#42-oob-divzero越界与除零扫描) 做一次扫描 |

## 1. 开始使用

### 1.1 连接模型

Room 需要可用的大模型（云端或本机服务）才能对话和生成文档。

**推荐（终端）：**

```text
room setup
```

按提示选择服务商并填写 API 密钥。本机模型若无需密钥，按提示跳过即可。

**已在界面内时：** 按 **Ctrl+M**，或输入 `/setup`。

配置保存在本机，与项目目录分离。完成后可用 `/model` 切换具体模型；窗口底部状态栏会显示当前模型名称。

### 1.2 进入项目再启动

Room 将**当前工作目录**视为项目根目录：

```text
cd 您的项目路径
room
```

请先进入真实项目目录再启动，以便助手正确感知路径与文件。

### 1.3 首次建议自检（可选）

```text
room doctor
```

若引擎、助手（agent）、技能（skills）、模型（model）均正常，即可开始使用。  
判读说明见 [附录 C](#附录-c-环境自检-room-doctor)。

## 2. 认识界面

```
┌ 标题栏 ──────────────────────────────────────────┐
│                                                  │
│     对话消息区                 │  进度 / 大纲     │
│     （提问与回复）             │  （可关闭）      │
│                                                  │
│     输入框 ─────────────────────────────────────│
├ 状态栏：快捷键提示 · 当前模型 ───────────────────┤
```

| 区域 | 作用 |
|------|------|
| **对话消息区** | 您的提问、助手回复与执行过程；可滚动，可选中复制 |
| **右侧栏** | 文档任务进度与大纲；不需要时可关闭以加宽对话区 |
| **输入框** | 自然语言，或输入 `/` 使用命令 |
| **状态栏** | 常用快捷键提示与当前模型 |

### 发送与滚动（体验说明）

| 现象 | 含义 |
|------|------|
| 发送后问题在**上方**，下面留白等回复 | 「本轮从问题开始读」 |
| 回复变长，列表**跟着最新内容**往下 | 自动跟读 |
| 您**手动上翻**后，不再硬拽回底部 | 浏览历史优先 |
| 任务未结束时再发送，消息**排队** | 不会丢，按序处理 |
| 短回复结束后下方仍有一段空白 | 内容保持顶对齐（不是故障） |

向上浏览足够久时，用于顶对齐的空白会逐渐被「吃掉」。若仍担心，见 [第 5 章](#5-常见情况)。

## 3. 日常操作

### 3.1 对话

在输入框输入问题，按 **Enter** 发送。可复制例句见 [示例集](#示例集-可以直接照着试)。

| 操作 | 方法 |
|------|------|
| 发送 | **Enter** |
| 换行（多行） | **Shift+Enter** |
| 复制 | 鼠标拖选后松开 |
| 展开过长内容 | **双击**该内容块 |
| 中断当前任务 | **Ctrl+C** |
| 清空对话显示 | `/clear` 或 **Ctrl+L** |

**小提示：** 尽量带上文件路径、报错原文或期望结果；需要「只分析不改动」时在问题里写清楚；上一条还在跑时可继续输入，消息会排队。

### 3.2 文档与扫描（入口）

- **技术文档（paper-derived）** — 样例模板派生结构化文档。完整步骤见 **[4.1](#41-paper-derived模板派生文档)**；中断后恢复见 **[4.1.2](#412-随时中断随时-continue-恢复)**。  
- **C 代码扫描（oob-divzero）** — 数组越界与除零专项检查。完整用法见 **[4.2](#42-oob-divzero越界与除零扫描)**。

### 3.3 命令菜单与侧栏

输入 **`/`** 打开命令列表（方向键选择，**Enter** 确认）。完整列表见 [附录 A](#附录-a-命令速查)。

| 需求 | 操作 |
|------|------|
| 切换或查看模型 | `/model`，或 **Ctrl+M** |
| 重新配置连接与密钥 | `/setup` 或 **Ctrl+M** |
| 显示 / 隐藏右侧栏 | **Ctrl+B**（macOS 多为 **⌘B**） |

## 4. Room 套件

Room 与下列能力**同一安装包交付**，在界面用 `/` 命令或自然语言调用即可（以 `room doctor` 检查结果为准）。

| 组件 | 解决什么 | 详见 |
|------|----------|------|
| **paper-derived** | 按样例模板派生结构化技术文档 | [4.1](#41-paper-derived模板派生文档) |
| **oob-divzero** | C 代码数组越界与除零专项扫描 | [4.2](#42-oob-divzero越界与除零扫描) |

**公共约定：** 在**项目目录**下启动 `room`；过程文件（如 `.pd`、`.oob-divzero`）落在该项目中；`/` 可查看命令与 Skills。

### 4.1 paper-derived：模板派生文档

#### 4.1.1 它是什么

**paper-derived** 是 Room 内置的文档引擎（套件中的 `paper-derived` / `paper-derived.exe`）：

> **按您提供的样例版式与结构，把项目资料「派生」成结构化技术文档。**

适合设计说明、方案说明书等有固定章节骨架的材料——不是从空白聊天里随便写一篇。日常用界面命令即可，不必手敲底层 CLI。

| 您要做的事 | 在 Room 里 |
|------------|------------|
| 注册 / 管理模板 | `/template …` |
| 开始生成 | `/new` |
| 中断后恢复 | **`/continue`**（见 [4.1.2](#412-随时中断随时-continue-恢复)） |
| 看任务状态 | `/status` |

#### 4.1.2 随时中断，随时 `/continue` 恢复

> **文档生成在任何时候被中断、打断或退出后，都可以再开 Room，用 `/continue` 从上次进度接着跑——不必从头 `/new`。**

| 情况 | 之后怎么做 |
|------|------------|
| 生成中 **Ctrl+C** / 取消 | **`/continue`** |
| 关掉终端、退出 Room（**Ctrl+Q**） | 同一项目再 `room` → **`/continue`** |
| 休眠、断网、进程被打断 | 同一项目 → **`/continue`** |
| 想先做别的事 | 直接停；回来 **`/continue`** |
| 不确定有没有未完成任务 | **`/status`**，有则 **`/continue`** |

进度写在**当前项目目录**（常见 `.pd`）。请在**同一路径**下启动 Room 再继续。

```text
cd 您的项目路径
room
/continue
```

仅当界面明确提示「无法恢复 / 没有可继续的任务」时，才需要 `/new`（或按提示清理会话）。**日常默认路径就是 `/continue`。**

#### 4.1.3 推荐工作流

**① 准备样例** — 优先 **`.docx` 或 `.md`**；旧 **`.doc`** 请先另存为 `.docx`。样例应能代表贵司真实版式。

**② 注册模板**

```text
/template register ./docs/样例-设计说明.docx 设计说明
```

- 最后一项是显示名称；列表 / 删除：`/template list`、`/template delete …`  
- 可选快速注册：`/template register --fast …`（以界面为准）

**③ 开始生成**

```text
/new
```

选模板 → 补资料 → 确认。对话区显示步骤，右侧「进度 / 大纲」跟章节推进。

**④ 中断后** — 见 [4.1.2](#412-随时中断随时-continue-恢复)，用 **`/continue`** 接着跑。

也可自然语言（命令路径更稳）：「用『设计说明』模板，根据 docs/需求说明.md 生成一版设计文档。」

#### 4.1.4 结果在哪里

| 类型 | 常见位置 |
|------|----------|
| 会话 / 中间状态 | 项目下的 **`.pd`**（及界面提示） |
| 交付文档 | 完成时界面给出的输出路径 |

### 4.2 oob-divzero：越界与除零扫描

#### 4.2.1 它是什么

**oob-divzero** 是 Room 集成的 **C 代码**专项检查（`oob-divzero` / `oob-divzero.exe` + 同名 Skill），专注：

1. **数组 / 缓冲区越界**（含常见指针算术访问）  
2. **除零 / 取模零**

```text
C 源码 → 召回可疑点 → AI 语义判定 → （可选）ASan 验证 → 分级报告
```

| 层 | 谁做 | 做什么 |
|----|------|--------|
| **召回** | 工具 | 找出 `arr[i]`、`/`、`%`、指针加减等 |
| **判定** | Room 中的 AI 助手 | 判断有无有效边界 / 非零检查 |
| **验证** | 工具 + sanitizer | 尝试触发，确认真实崩溃 |

#### 4.2.2 在 Room 里怎么用

| 方式 | 做法 |
|------|------|
| **斜杠命令**（推荐写清目标） | `/oob-divzero` · `/oob-divzero path/to/file.c` · `/oob-divzero src/` |
| **自然语言** | 「帮我扫一下 src/driver/ 的越界和除零」 |
| **Skill 入口** | `/skill oob-divzero 扫描 ./src`（以 `/` 菜单实际名称为准） |

不带路径时，助手会先说明能力并请您指定目标。请在含目标代码的**项目目录**下启动 Room。

#### 4.2.3 报告怎么读

| 级别 | 含义 | 您该怎么看 |
|------|------|------------|
| **Confirmed** | 验证层触发了问题（如 ASan） | 优先修 |
| **Likely** | 判定可疑，未自动复现 | 人工再审 |
| **Defensive** | 缺守卫但当前路径可能仍安全 | 加固建议，优先级较低 |

过程与报告一般在：

```text
.oob-divzero/<时间戳>/
  pending.json
  judged.json
  oob-divzero-report.html   ← 可用浏览器打开
```

#### 4.2.4 能力边界

**覆盖较好：** 同函数内数组访问与除法；宏守卫、类型转换惯用法；简单指针算术。  

**通常较弱：** 远距离跨函数守卫；大小未知的堆缓冲；完整 C++ 验证；「所有安全问题」——**不是**通用 SAST 替代品。

#### 4.2.5 环境与 doctor

| 检查 | 说明 |
|------|------|
| `room doctor` 中 oob 正常 | 扫描 CLI 可用 |
| 编译工具链 / ASan 缺失 | 仍可能召回 + 判定；Confirmed 验证受限 |
| 「不是有效的 Win32 应用程序」 | 二进制与系统不匹配，需重装正确套件 |

## 5. 常见情况

### 能打开界面，但无法回答

多半是模型未配置或密钥无效：`room setup` / **Ctrl+M** → `/model` 确认 → `room doctor` 查看 model、agent。

### 模板注册提示已存在

`/template` 查看列表：删除旧模板，或换显示名称后重新注册。

### 旧版 Word（.doc）注册异常

请先另存为 **`.docx`** 再注册。

### 右侧栏占用空间

**Ctrl+B**（macOS：**⌘B**）收起。

### 需要停止正在执行的任务

**Ctrl+C** 或退出 Room。  
**文档生成：** 停完后在同一项目用 **`/continue` 恢复**（见 [4.1.2](#412-随时中断随时-continue-恢复)）。普通对话一般重新发消息即可。

### 已升级软件，界面仍无变化

需**重新运行新版本安装程序**；仅更新说明文档不会改程序。装完后 `room --version` 核对。

### doctor 显示 oob 或 paper-derived 异常

- **paper-derived** 找不到 / 版本不对：与 Room 成对的完整套件  
- **oob-divzero** 无法运行：多为架构或损坏的二进制，联系供应商重装  
- **仅缺编译工具链：** 对话与文档通常仍可用；oob 的 Confirmed 验证变弱  

### 发送后底部一大片空白，是不是坏了？

通常不是。短回复**顶对齐**会留下空白；向上浏览后空白会减少。见 [§2 体验说明](#发送与滚动体验说明)。

## 附录 C. 环境自检（room doctor）

```text
room doctor
```

| 检查结果 | 通常含义 | 建议 |
|----------|----------|------|
| 引擎 / agent / skills / model 正常 | 可正常对话与写文档 | — |
| 模型未配置 | 无法对话或生成 | `room setup` 或 **Ctrl+M** |
| 文档转换器未找到 | 主要影响旧 **.doc** | 样例另存为 **.docx** |
| 组件无法运行 / 架构提示 | 安装包不匹配或损坏 | 向供应商重装 |
| 编译工具链缺失 | 影响部分深度验证 | 日常对话与写文档通常不受影响 |

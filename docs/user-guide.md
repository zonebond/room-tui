# Room 使用手册

**Room** 是安装在您电脑上的本地智能工程助手。  
在项目文件夹中打开后，您可以：用自然语言完成开发相关任务，并基于样例模板生成结构化技术文档。

本手册面向**最终使用客户**。按顺序阅读即可完成安装、配置与日常使用。

---

## 1. 产品能做什么

| 能力 | 说明 |
|------|------|
| **智能对话** | 在项目目录中提问、分析代码、执行必要的本地操作 |
| **文档生成** | 根据样例模板与项目资料，派生规范类技术文档 |
| **任务可视** | 界面右侧显示生成进度与文档大纲，过程可感知 |
| **本地优先** | 程序与密钥配置在本机；工作内容以您打开的项目目录为准 |

> Room 通过您配置的大模型服务（云端或本机）工作。首次使用需完成一次模型连接配置。

---

## 2. 运行环境

- **Windows 10/11**（64 位），或 **macOS**
- 稳定的网络（使用云端模型时）
- 安装包由供应商提供（安装程序或压缩包）

请使用与电脑架构匹配的安装包（常见为 64 位 x86；若为 ARM 电脑请使用对应版本）。

---

## 3. 安装

### 3.1 Windows

1. 获取供应商提供的 **Room 安装程序**（`Room-Setup-….exe`）或套件压缩包。  
2. **安装程序**：双击按向导完成安装。  
3. **压缩包**：解压后运行其中的 `install.bat` 或 `install.ps1`。  
4. **关闭并重新打开**命令提示符 / PowerShell / 终端。  
5. 输入以下命令确认：

```text
room --version
```

能显示版本号即表示安装成功。

**若提示找不到 `room` 命令**  
请重新打开终端后再试；或联系供应商确认安装是否将程序目录加入了系统路径。

### 3.2 macOS

1. 解压供应商提供的套件。  
2. 在终端进入解压目录，执行：

```bash
chmod +x install.sh
./install.sh
```

3. 若提示找不到命令，请将 `~/.local/bin` 加入 PATH（写入 `~/.zshrc` 后执行 `source ~/.zshrc`）。  
4. 执行 `room --version` 确认。

## 4. 首次配置（连接模型）

安装完成后，请先配置大模型，否则无法对话或生成文档。

**方式一（推荐）** — 在终端执行：

```text
room setup
```

按提示选择服务商并填写 API 密钥（本机模型若无需密钥，可按提示留空）。

**方式二** — 打开 Room 后：

- 按 **Ctrl+M**，或  
- 输入 `/setup`

配置保存在本机用户目录下的 Room 专用配置中，**不会**写入您的项目源码。

配置完成后，建议执行：

```text
room doctor
```

用于检查引擎、模型与组件是否就绪（详见第 9 节）。

## 5. 启动与工作方式

### 5.1 打开项目

Room 以**当前文件夹**为工作项目：

```text
cd 您的项目路径
room
```

请先进入真实项目目录再启动，以便助手正确感知文件与路径。

### 5.2 界面说明

| 区域 | 作用 |
|------|------|
| **中间消息区** | 您的提问、助手回复、执行过程；可滚动、拖选复制 |
| **右侧栏** | 文档任务进度与大纲（可关闭以扩大对话区） |
| **底部输入框** | 输入问题或命令 |
| **最底状态栏** | 快捷键提示与当前模型名称 |

### 5.3 基本操作

| 操作 | 方法 |
|------|------|
| 发送 | **Enter** |
| 换行（多行输入） | **Shift+Enter** |
| 复制内容 | 鼠标拖选后松开（自动复制到剪贴板） |
| 展开过长内容 | **双击**该内容块 |
| 显示/隐藏右侧栏 | **Ctrl+B**（Mac 上多为 **Command+B**） |
| 切换或配置模型 | **Ctrl+M** |
| 中断当前任务 | **Ctrl+C** |
| 退出 | **Ctrl+Q** |

发送后，您的问题会显示在对话上方；回复较长时，列表会自动跟随最新内容。若您手动向上翻阅历史，则不会强行跳回底部。

## 6. 使用场景

### 6.1 智能对话

在输入框直接输入自然语言，例如：

- 「介绍一下当前项目结构」  
- 「帮我解释这段报错」  
- 「列出目录中的主要文件」  

**Enter** 发送即可。  
若上一个任务尚未结束，新消息会自动排队，完成后按顺序处理，无需担心丢失。

### 6.2 生成技术文档

典型步骤：

**① 准备样例**  
准备一份可作为格式/结构参考的样例文件。  
**建议使用 `.docx` 或 `.md`。** 若只有旧版 `.doc`，请先用 Word 或 WPS **另存为 `.docx`**。

**② 注册模板**

```text
/template register ./样例文件.docx 模板显示名称
```

**③ 开始生成**

```text
/new
```

按界面提示选择模板与资料，确认后开始生成。右侧可查看进度。

**④ 中断后续跑**

```text
/continue
```

生成结果与过程数据位于项目目录下的相关文件夹中（通常包含 `.pd` 目录）；具体文件名以界面提示为准。

### 6.3 常用命令

在输入框输入 **`/`** 可打开命令菜单。

| 命令 | 说明 |
|------|------|
| `/new` | 新建文档生成任务 |
| `/continue` | 继续未完成的任务 |
| `/status` | 查看任务状态 |
| `/template` | 管理模板（列表、注册、删除等） |
| `/model` | 查看或切换模型 |
| `/setup` | 重新配置模型连接 |
| `/clear` | 清空当前对话显示 |
| `/help` | 显示帮助 |

部分环境还提供专项技能（例如代码安全检查类技能），以您安装包中实际提供的为准；输入 `/` 可在菜单中查看。

## 8. 环境自检：`room doctor`

在终端执行：

```text
room doctor
```

用于检查本机 Room 是否工作正常。

### 8.1 正常时应关注的项

| 检查项 | 正常表现 |
|--------|----------|
| 引擎（paper-derived） | 已找到，版本信息正常 |
| room agent | ok |
| skills | ok，必要技能已加载 |
| model | 已显示当前模型 |

上述项正常，即可进行对话与文档生成。

### 8.2 常见提示说明

| 提示 | 对您的影响 | 建议处理 |
|------|------------|----------|
| 模型未配置 | 无法对话/生成 | 执行 `room setup` 或 **Ctrl+M** |
| 找不到命令 `room` | 无法启动 | 新开终端；确认安装完整；联系供应商 |
| 文档转换器（LibreOffice）未找到 | 仅影响**旧版 .doc** | 将样例另存为 **.docx** 再注册 |
| 某组件「不是有效的 Win32 应用程序」 | 对应组件无法运行 | 安装包与系统架构不匹配或文件损坏，请向供应商重新获取安装包并重装 |
| 编译工具链未找到 | 仅影响部分深度代码验证功能 | 一般对话与文档不受影响；需要时联系供应商获取完整版 |

将 `room doctor` 的完整输出提供给供应商，可加快技术支持响应。

## 9. 常见问题

**Q：安装后提示找不到 room？**  
A：请关闭终端后重新打开再试。Windows 请确认使用「用户」环境中的命令行。仍不行请联系供应商协助检查安装路径。

**Q：界面能打开，但无法回答问题？**  
A：请执行 `room setup` 配置有效的模型与密钥，再用 `room doctor` 查看 model 与 agent 是否正常。

**Q：模板注册提示已存在？**  
A：使用 `/template` 查看列表，删除旧模板或更换新的显示名称后重新注册。

**Q：旧 Word 文档（.doc）注册效果不好？**  
A：请先另存为 **.docx** 后再注册，兼容性与稳定性更好。

**Q：供应商说升级了，但我这边界面没变化？**  
A：Windows 等环境使用的是已安装的程序文件，需要**重新运行新的安装程序完成升级**，仅替换文件夹中的说明文档不会更新程序。升级后请新开终端并执行 `room --version` 核对版本。

**Q：数据与密钥安全吗？**  
A：密钥保存在本机用户配置目录，不进入您的项目源码目录。项目相关过程文件在项目目录内生成，请按贵司规范进行备份与保密。

## 11. 使用前检查清单

1. 已安装 Room，且 `room --version` 有输出  
2. 已完成 `room setup`  
3. `room doctor` 中引擎、agent、skills、model 正常  
4. 已进入实际项目目录再执行 `room`  
5. 需要生成文档时：已用 `/template register` 注册样例，并用 `/new` 启动任务

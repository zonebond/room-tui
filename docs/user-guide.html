Foreword

前言

这份手册是什么

Room 装在您的电脑上，围绕当前打开的项目目录工作。配置好模型后，您可以：

  • 用自然语言完成开发相关问答与辅助操作
  • 按样例模板派生结构化技术文档
  • 在界面中查看任务进度与文档大纲

密钥与连接配置保存在本机用户目录，不会写入项目源码。

Chapter 01

开始使用

1.1连接模型

Room 需要可用的大模型（云端或本机服务）才能对话和生成文档。

推荐（终端）：

room setup

按提示选择服务商并填写 API 密钥。本机模型若无需密钥，按提示跳过即可。

已在界面内时：

  • 按 Ctrl+M，或
  • 输入 /setup

配置保存在本机，与项目目录分离。完成后可用 /model 切换具体模型； 窗口底部状态栏会显示当前模型名称。

1.2进入项目再启动

Room 将当前工作目录视为项目根目录：

cd 您的项目路径
room

请先进入真实项目目录再启动，以便助手正确感知路径与文件。 在错误目录中打开时，回答往往缺少项目上下文。

1.3首次建议自检（可选）

room doctor

若引擎、助手（agent）、技能（skills）、模型（model）均正常，即可开始使用。 判读说明见 附录 C。

发送与滚动（体验说明）

  • 发送后，您的问题会置于对话区上方，下方留给回复。
  • 回复变长时，列表会跟随最新内容向下滚动。
  • 若您手动向上翻阅历史，不会被强制拉回底部。
  • 上一任务未结束时，新消息会自动排队，按顺序处理，无需担心丢失。
Chapter 03

日常操作

3.1对话

在输入框输入问题，按 Enter 发送。例如：

  • 「这个项目整体结构是怎样的？」
  • 「这段报错是什么意思？」
  • 「帮我列一下主要源码目录」
操作方法
发送Enter
换行（多行）Shift+Enter
复制鼠标拖选后松开（通常进入系统剪贴板）
展开过长内容双击该内容块，可展开或收起
中断当前任务Ctrl+C
清空对话显示/clear 或 Ctrl+L

3.2生成技术文档

典型路径如下。

  1. 准备样例

    作为格式与章节结构的参考。优先使用 .docx 或 .md。 若仅有旧版 .doc，请先用 Word 或 WPS 另存为 .docx 再注册，兼容性更好。

  2. 注册模板 
    /template register ./样例文件.docx 模板显示名称
  3. 开始生成 
    /new

    按界面提示选择模板、补充资料并确认。右侧「进度 / 大纲」会同步更新。

  4. 中断后继续 
    /continue

    生成结果与过程数据通常位于项目目录相关文件夹中（常见包含 .pd 目录）； 具体路径以界面提示为准。

3.3使用命令菜单

在输入框输入 /，可打开命令列表：方向键选择，Enter 确认。 完整列表见 附录 A。 若安装包中包含专项技能（例如代码检查类），也会出现在 / 菜单中，以您机器上实际显示为准。

3.4切换模型与侧栏

需求操作
切换或查看模型/model，或 Ctrl+M
重新配置连接与密钥/setup 或 Ctrl+M
显示 / 隐藏右侧栏Ctrl+B（macOS 多为 ⌘+B）
Chapter 04

常见情况

能打开界面，但无法回答

多半是模型未配置或密钥无效。

  1. 执行 room setup，或在界面中使用 Ctrl+M / /setup
  2. 用 /model 确认当前模型
  3. 在终端执行 room doctor，查看 model、agent 是否正常

模板注册提示已存在

使用 /template 查看列表：删除旧模板，或更换显示名称后重新注册。

旧版 Word（.doc）注册异常

请先另存为 .docx 再注册。旧格式依赖本机转换环境，.docx 更稳定。

右侧栏占用空间

按 Ctrl+B（macOS：⌘+B）收起；需要查看进度时再打开。

需要停止正在执行的任务

按 Ctrl+C。文档任务中断后，多数情况可用 /continue 继续。

已升级软件，界面仍无变化

程序以安装到系统中的文件为准，需重新运行新版本安装程序完成升级。 仅更新说明文档不会改变程序本身。升级后请新开终端，执行 room --version 核对版本号。

Appendix C

环境自检（room doctor）

在终端执行：

room doctor
检查结果通常含义建议
引擎 / agent / skills / model 正常 对话与文档生成可正常使用 —
模型未配置 无法对话或生成 执行 room setup 或 Ctrl+M
文档转换器未找到 主要影响旧版 .doc 样例另存为 .docx 后注册
组件无法运行 / 架构相关提示 安装包与系统不匹配或文件损坏 向供应商重新获取安装包并重装
编译工具链缺失 仅影响部分深度代码验证 日常对话与写文档通常不受影响

将完整 room doctor 输出提供给支持人员，可加快响应。

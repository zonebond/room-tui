Foreword

前言

这份手册是什么

Room 装在您的电脑上，围绕当前打开的项目目录工作。配置好模型后，您可以：

  • 用自然语言完成开发相关问答与辅助操作
  • 按样例模板派生结构化技术文档
  • 在界面中查看任务进度与文档大纲

密钥与连接配置保存在本机用户目录，不会写入项目源码。

产品特色一览

Room 是嵌在项目目录里的工程助手，日常最能感知的差异：

Local

本地 · 以项目为中心

在哪个文件夹启动，就围绕哪个项目；密钥在本机用户目录。

忙时消息排队

4.1paper-derived：模板派生文档

4.1.1它是什么

按样例版式与结构，把项目资料「派生」成结构化技术文档。

适合设计说明、方案类材料。日常用界面命令即可。

您要做的事在 Room 里
注册 / 管理模板/template …
开始生成/new
中断后恢复/continue（4.1.2）
任务状态/status

4.1.2随时中断，随时 /continue 恢复

文档生成在任何时候被中断、打断或退出后，都可以再开 Room，用 /continue 接着跑——不必从头 /new。

情况之后怎么做
Ctrl+C / 取消/continue
关终端、退出 Room同一项目 room → /continue
休眠、断网、进程打断同一项目 → /continue
想先做别的事直接停；回来 /continue
不确定有无未完成任务/status，有则 /continue

进度在当前项目（常见 .pd）。请在同一路径下继续。

cd 您的项目路径
room
/continue

仅当提示「无法恢复」时才 /new。日常默认就是 /continue。

4.1.3推荐工作流

  1. 准备样例

    优先 .docx / .md；旧 .doc 先另存为 .docx。

  2. 注册模板 
    /template register ./docs/样例-设计说明.docx 设计说明

    /template list · /template delete … · 可选 --fast

  3. 开始生成 
    /new

    选模板 → 补资料 → 确认；右侧进度 / 大纲同步。

  4. 中断后

    见 4.1.2，用 /continue。

也可自然语言（命令更稳）：「用『设计说明』模板，根据 docs/需求说明.md 生成一版设计文档。」

类型常见位置
会话 / 中间状态项目下 .pd
交付文档完成时界面给出的路径

4.2oob-divzero：越界与除零扫描

4.2.1它是什么

专注 C 代码的数组/缓冲区越界与除零/取模零：

C 源码 → 召回 → AI 判定 → （可选）ASan 验证 → 分级报告
层谁做做什么
召回工具找出访问与除法检查点
判定AI 助手有无有效边界/非零检查
验证工具 + sanitizer尝试触发崩溃

4.2.2在 Room 里怎么用

方式做法
斜杠命令/oob-divzero · /oob-divzero file.c · /oob-divzero src/
自然语言「帮我扫一下 src/driver/ 的越界和除零」
Skill/skill oob-divzero 扫描 ./src

请在含目标代码的项目目录下启动 Room。

4.2.3报告怎么读

级别含义怎么看
Confirmed验证层触发优先修
Likely可疑未复现人工再审
Defensive缺守卫但可能仍安全加固建议
 
.oob-divzero/<时间戳>/
  pending.json · judged.json · oob-divzero-report.html

4.2.4能力边界

较好：同函数内访问与除法、宏守卫、简单指针算术。
较弱：远距离守卫、未知堆大小、完整 C++——不是通用 SAST。

4.2.5环境与 doctor

检查说明
doctor 中 oob 正常CLI 可用
缺工具链 / ASan仍可召回+判定；Confirmed 受限
非有效 Win32 应用架构不匹配，重装套件
Chapter 05

常见情况

能开界面但不回答

room setup / Ctrl+M → /model → room doctor。

模板已存在

/template 列表后删除或换名注册。

旧 .doc 注册异常

先另存为 .docx。

侧栏占空间

Ctrl+B 收起。

停止任务 / 文档中断后

Ctrl+C 或退出。文档生成：同一项目 /continue（4.1.2）。

升级后界面没变

需重跑安装程序；用 room --version 核对。

doctor 报 oob / paper-derived 异常

引擎需与 Room 成对完整套件；oob 二进制架构要匹配；仅缺工具链时日常对话/文档通常仍可用。

底部一大片空白？

通常是短回复顶对齐，不是故障。见 §2。
